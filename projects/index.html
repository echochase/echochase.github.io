Projects

Throughout my time at UNSW, I've worked on many personal projects in my spare time. Here are some of the more ambitious ones.

Intrusion

This is a personal project I am currently working on with a group of friends. Intrusion is a card game that models different cyber-attack and defense scenarios on a vulnerable system. I created this game at the end of last year, and its purpose is to introduce the public to the world of cyber security, regardless of their background (or lack thereof) in computer science.

We're developing this project with a slightly bigger team of 6 developers and 3 artists. We're using Java for the backend, and React for the frontend. My friend Jerry and I are working on the backend, and my friends Shaam, Sohum and Manav working on the frontend. Finally, my friend James is doing the heavy lifting with figuring out the server logic, i.e. how in the world we're going to integrate Java Socketio into our codebase (seriously, the socketio implementations need better documentation, and we don't want to use the Java Socket Library unless we absolutely have to).

Right now we have about all of the backend game logic done, and the frontend is making significant headway. The art is also halfway done, since this project is thankfully less art-heavy than Machiavellian Pursuits. We're thinking to deploy this project ASAP, though it depends how busy everyone is once term starts. Either way, we already have a lot to show for it!

ICPC Connect

ICPC Connect is a team builder platform for the International Collegiate Programming Contest, a prestigious competitive programming competition for more than 50,000 university students around the world. Our clients are the ICPC executives for the South Pacific, including a UNSW Site Coordinator.

In developing this project, I worked in a team of 6 to create a service that connects students without teams, reducing the workload for coaches who previously had to form teams manually. We also provided an interface that makes it easy for coaches and site coordinators to view and manage all the registered teams.

As the Product Owner of this project, I was responsible for facilitating the effective communication between ICPC administrators and my team, ensuring our product meets client needs and expectations. I organized stakeholder meetings at the start of each development sprint to receive client feedback and ensure alignment on priorities and requirements.

As one of two frontend developers, I designed the user interface using Figma, helping to optimise the user experience and flow. We then faithfully implemented our frontend design using Typescript, React, Next.js and Material UI.

Machiavellian Pursuits

Just a passion project I do on the side.

Machiavellian Pursuits is a card game where players battle for victory through manipulation and unstable alliances. Having previously created the game myself, I first printed this set of cards out to play with a group of friends in person, but we quickly realized that it made for an ambitious but fulfilling programming project. After we clarified the rules and beta-tested the game, we took to programming and decided on using ReactJS for the frontend, and TypeScript for the backend.

We're a team of five friends, 3 of which (including myself) are developers and 2 are artists. This endeavour has taught me to think like a leader and project manager as well as a developer; as the creator and being most familiar with the details of the game, I am responsible for directing the creative process as well as specifying the requirements on the technical side. As of right now, the game is incomplete. However, we have completed implementation of rooms (sessions) using Socket.io, made significant headway on the graphics and backend, and have finished documentation (including rules and gameplay guidelines) and plans for user interface and game logic. When the game is finished, we will deploy it publicly on a web server where it can be played and enjoyed from anywhere.
